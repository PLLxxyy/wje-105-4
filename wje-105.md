## 一、硬性约束

### 1.1 构建可运行性
- 项目使用 Vite + Vue 3 + TypeScript，生成后必须能通过 `npm install && npm run dev` 无报错启动
- 所有 TypeScript 类型声明必须正确，不得出现 `any` 泛滥或 `@ts-ignore` 屏蔽
- Tailwind CSS 必须正确配置（tailwind.config.js / postcss.config.js），样式能正常渲染
- Pinia store 必须在 main.ts 中正确注册，router 必须正确挂载
- 所有页面路由必须可达，不得出现空白页或死链

### 1.2 功能完整性
- 每个页面/组件必须实现其描述的完整功能，不得留 TODO / 占位符 / mock-return
- 所有 CRUD 操作必须真正写入 localStorage 并在刷新后保持
- 调制动画必须可播放且视觉效果完整（分层倒入、液面晃动）
- 配方对比功能必须能同时展示两个配方并高亮差异
- 筛选功能必须能实际过滤数据，不是纯 UI 装饰

### 1.3 代码质量
- 每个文件只承担单一职责，严禁将多个不相关逻辑塞入同一文件
- 组件 props 必须使用 TypeScript interface 定义，emits 必须声明类型
- 所有共享枚举/常量必须从 `constants/enums.ts` 导入，不得在组件内硬编码魔法字符串
- localStorage 读写必须通过 `useLocalStorage` hook 封装，不得在组件中直接调用 `window.localStorage`
- 错误边界必须捕获 localStorage 读写异常并给出友好提示，不得让应用白屏崩溃

## 二、项目主要需求

### 2.1 项目概述

**项目名**：调酒配方实验室
**英文短名（COMPOSE_PROJECT_NAME）**：`wjecocktaillab`

构建一个纯前端的鸡尾酒配方管理与演示平台。用户可以浏览经典鸡尾酒配方、创建自定义配方、维护个人材料库、观看分层调制动画演示，并通过收藏夹管理喜爱的配方。所有数据持久化至浏览器 localStorage。

### 2.2 数据模型

#### CocktailRecipe（配方）
```typescript
interface CocktailRecipe {
  id: string;                          // UUID
  name: string;                        // 配方名称
  description: string;                 // 描述
  baseSpirit: string;                  // 基酒名称
  glassType: GlassType;                // 杯型枚举
  method: MixMethod;                   // 调制方法枚举
  garnish: string;                     // 装饰物描述
  difficulty: 'easy' | 'medium' | 'hard'; // 难度
  isAlcoholic: boolean;                // 是否含酒精
  ingredients: RecipeIngredient[];     // 材料列表
  steps: string[];                     // 调制步骤
  tags: string[];                      // 风格标签（清爽/浓郁/果味/经典等）
  createdAt: number;                   // 创建时间戳
  updatedAt: number;                   // 更新时间戳
}
```

#### RecipeIngredient（配方材料条目）
```typescript
interface RecipeIngredient {
  id: string;           // UUID
  ingredientId: string; // 关联 Ingredient.id
  amount: number;       // 用量
  unit: 'ml' | 'oz' | 'dash' | 'drop' | 'piece'; // 单位
}
```

#### Ingredient（材料）
```typescript
interface Ingredient {
  id: string;                          // UUID
  name: string;                        // 材料名称
  category: IngredientCategory;        // 分类枚举
  abv: number;                         // 酒精度百分比（0-100）
  description: string;                 // 描述
  color: string;                       // HEX 颜色值，用于动画分层显示
}
```

#### Collection（收藏夹）
```typescript
interface Collection {
  id: string;            // UUID
  name: string;          // 收藏夹名称
  recipeIds: string[];   // 收藏的配方 ID 列表
  createdAt: number;     // 创建时间戳
}
```

### 2.3 核心页面

1. **首页（Home）**：展示推荐配方卡片、快速入口（创建配方/材料库/动画演示/收藏夹）、最近浏览记录
2. **配方浏览（RecipeList）**：卡片式网格展示所有配方，支持按基酒/酒精度范围/风格标签/难度筛选，支持搜索
3. **配方详情（RecipeDetail）**：完整展示配方信息、材料用量表、调制步骤、杯型图标、装饰物描述，可跳转动画演示
4. **配方创建（RecipeCreate）**：表单编辑器——填写名称/描述/基酒/杯型/方法/难度/装饰物，动态增删材料行（选择材料+用量），调制步骤编辑器
5. **配方编辑（RecipeEdit）**：复用创建表单，加载已有数据并保存修改
6. **材料库（IngredientLibrary）**：展示所有材料（含预置+自定义），按分类筛选，可新增/编辑/删除自定义材料
7. **收藏夹（Collections）**：创建收藏夹、将配方加入/移出收藏夹、按收藏夹分组查看
8. **动画演示（AnimationDemo）**：选择配方后播放调制动画——分层倒入效果（每层颜色对应材料颜色）、液面晃动、杯型轮廓，支持暂停/重播
9. **配方对比（RecipeCompare）**：左右分栏展示两个配方，高亮材料和比例差异

### 2.4 横切关注点

#### 2.4.1 主题切换（亮/暗模式）
- **实现层**：Pinia `useThemeStore` 管理当前主题状态（`light` / `dark`），持久化至 localStorage
- **触达层**：Tailwind CSS `dark:` 前缀 + CSS 变量（`--color-bg`, `--color-text`, `--color-card` 等）
- **组件层**：`<ThemeToggle>` 切换按钮，`<html>` 标签切换 `dark` class
- 所有页面和组件必须适配亮/暗两种模式

#### 2.4.2 全局错误处理
- **ErrorBoundary 组件**：捕获子组件渲染错误，展示友好错误页面而非白屏
- **localStorage 异常处理**：`useLocalStorage` hook 内部 try-catch，存储满/不可用时提示用户
- **配方数据校验**：保存配方时校验必填字段（名称、至少一种材料、杯型、方法），不合法则高亮错误字段

### 2.6 共享组件

#### `<GlassIcon>`
- **功能**：根据 `GlassType` prop 渲染对应杯型的 SVG 图标
- **props**：`type: GlassType`, `size?: number`, `color?: string`
- **使用位置**：配方卡片、配方详情、配方编辑器预览

#### `<IngredientTag>`
- **功能**：展示材料名称标签，背景色使用材料的 `color` 属性，文字自动计算对比色
- **props**：`ingredient: Ingredient`, `amount?: number`, `unit?: string`, `closable?: boolean`
- **使用位置**：配方详情材料列表、配方编辑器材料选择区、材料库列表

#### `<EmptyState>`
- **功能**：空状态占位图 + 提示文案 + 可选操作按钮
- **props**：`icon?: string`, `title: string`, `description?: string`, `actionText?: string`, `actionHandler?: () => void`
- **使用位置**：配方列表无数据、收藏夹为空、材料库为空

#### `<FilterBar>`
- **功能**：通用筛选栏，包含基酒下拉、酒精度滑块、难度选择、风格标签多选、搜索框
- **props**：`filters: FilterState`, `onChange: (filters: FilterState) => void`
- **使用位置**：配方浏览页、材料库页（裁剪掉不适用的筛选项）

#### `<ThemeToggle>`
- **功能**：亮/暗模式切换按钮，带动画过渡
- **使用位置**：全局导航栏

### 2.7 共享 Hooks

#### `useLocalStorage`
- **功能**：封装 localStorage 读写，支持泛型、默认值、JSON 序列化/反序列化、读写异常捕获
- **签名**：`useLocalStorage<T>(key: string, defaultValue: T): Ref<T>`
- **使用位置**：所有 Pinia store 的数据持久化

#### `useFilter`
- **功能**：通用筛选逻辑，接收数据数组和筛选条件，返回筛选后结果
- **签名**：`useFilter<T>(items: Ref<T[]>, filterFn: (item: T, filters: F) => boolean): { filtered: ComputedRef<T[]>, filters: Ref<F>, resetFilters: () => void }`
- **使用位置**：配方浏览页筛选、材料库筛选

#### `useAnimation`
- **功能**：调制动画控制逻辑——播放/暂停/重播、当前步骤、进度百分比
- **签名**：`useAnimation(recipe: CocktailRecipe, ingredients: Ingredient[]): { play: () => void, pause: () => void, reset: () => void, progress: Ref<number>, currentStep: Ref<number>, isPlaying: Ref<boolean> }`
- **使用位置**：动画演示页

## 四、核心流程

### 流程一：浏览配方
1. 用户进入首页，看到推荐配方卡片网格
2. 点击"浏览全部"进入配方浏览页，顶部显示 `<FilterBar>`
3. 用户通过基酒下拉、酒精度滑块、难度选择、风格标签筛选配方
4. 点击任意配方卡片跳转至配方详情页，查看完整配方信息
5. 在详情页可点击"调制演示"跳转动画演示页

### 流程二：创建配方
1. 用户点击"创建配方"进入表单页
2. 填写配方名称、描述、选择基酒、杯型、调制方法、难度
3. 动态添加材料行——从材料库下拉选择材料，输入用量和单位
4. 添加调制步骤（可动态增减步骤）
5. 系统实时估算酒精度（根据各材料 ABV 和用量加权计算）
6. 点击保存，数据校验通过后写入 Pinia store 和 localStorage

### 流程三：调制动画演示
1. 用户在配方详情或动画演示页选择一个配方
2. 动画组件根据配方的 `method` 属性选择动画序列
3. 分层倒入动画：逐层倒入每种材料，液面颜色对应材料的 `color` 属性
4. 倒入过程中液面有轻微晃动效果（CSS animation 或 Canvas）
5. 完成后展示成品效果，支持暂停/重播

### 流程四：收藏管理
1. 用户在配方详情或配方卡片上点击收藏按钮
2. 首次收藏时可选择已有收藏夹或创建新收藏夹
3. 在收藏夹页面查看所有收藏夹，展开查看收藏的配方列表
4. 支持将配方从收藏夹移除、删除收藏夹

## 六、代码存放规则

这是纯前端单页应用（SPA），无后端服务、无数据库。

所有前端代码统一放在 `frontend/` 目录下：

```
wjecocktaillab/
├── frontend/           # 前端项目根目录
│   ├── src/            # 源码（见第三章文件结构）
│   ├── public/         # 静态资源
│   ├── package.json
│   └── ...配置文件
├── .gitignore
└── README.md
```

**数据存储方案**：
- 所有用户数据（配方、材料、收藏夹、主题偏好）存储于浏览器 localStorage
- 通过 `hooks/useLocalStorage.ts` 统一封装读写，内部做 JSON 序列化/反序列化
- 通过 `utils/storage.ts` 定义所有 localStorage 键名常量（避免硬编码散落各处）
- 首次启动时自动初始化预置材料和配方数据
